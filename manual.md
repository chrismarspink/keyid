## 1. 개요

KeyID는 브라우저에서 동작하는 PKI(공개키 기반 구조) 디지털 신원 관리 앱입니다.
외부 서버나 CA(인증 기관) 없이 **기기 내부에서만** 키를 생성하고 보관합니다.

### 주요 기능

| 기능 | 설명 |
|------|------|
| 신원 생성 | X.509 자체 서명 인증서 + ECDSA P-256 키 쌍 생성 |
| 파일 서명 | CMS SignedData(detached) 방식으로 파일에 디지털 서명 |
| 서명 검증 | `.pkis-sig` 파일과 원본 파일로 서명 유효성 확인 |
| 파일 암호화 | CMS EnvelopedData + ECDH 키 합의 방식으로 지정 수신자에게 암호화 |
| 파일 복호화 | 자신의 개인 키로 암호화된 파일 복호화 |
| 연락처 관리 | 다른 사람의 인증서를 연락처로 등록 |
| 신원 갱신 | 인증서 유효기간 연장 또는 새 키 발급 |

### 보안 특징

- **개인 키는 기기 밖으로 절대 나가지 않습니다** — 모든 암호화 연산은 브라우저 내부에서만 수행
- 개인 키는 WebAuthn PRF(생체인증) 또는 PBKDF2(비밀번호)로 AES-GCM 암호화되어 IndexedDB에 저장
- ECDSA P-256 / SHA-256 서명, ECDH P-256 키 합의, AES-256-GCM 암호화 사용

## 2. 설치 및 실행

### 개발 서버 실행

```bash
cd keyid
npm install
npm run dev -- --host
```

기본 접속 주소: `http://localhost:5173`

```bash

```

## 3. 신원 생성

앱을 처음 실행하면 신원 생성 마법사가 표시됩니다.

### 단계 1: 기본 정보 입력

| 항목 | 설명 | 필수 |
|------|------|------|
| 이름 | 인증서의 CommonName (표시 이름) | ✅ |
| 이메일 | 인증서 이메일 필드 및 SubjectAltName | ❌ |
| 소속/기관 | 인증서 Organization 필드 | ❌ |
| 국가 코드 | 2자리 ISO 국가 코드 (예: KR, US) | ❌ |

### 단계 2: 키 보호 방법 선택

#### 생체 인증 (WebAuthn PRF) — 권장
- 기기의 지문/Face ID로 개인 키를 보호
- PRF(Pseudo-Random Function) 확장을 통해 AES 키 파생
- 지원 여부는 기기/브라우저에 따라 다름

> 선택 시 **백업 비밀번호**를 함께 설정할 수 있습니다.
> 생체인증 기기가 없거나 기기를 교체할 때를 대비한 대안 인증 수단입니다.

#### 비밀번호 (PBKDF2)
- 8자 이상의 비밀번호로 개인 키 보호
- PBKDF2(SHA-256, 반복 100,000회) → AES-256-GCM 암호화

### 생성 완료

생성되는 항목:
- X.509 v3 자체 서명 인증서 (유효기간 3년)
- ECDSA P-256 키 쌍 (공개키 + 봉인된 개인 키)
- SHA-256 지문 (인증서 식별자)

## 5. 신원 관리

사이드바 → **신원** (또는 `/identity`)

### 인증서 정보 조회

- 이름, 이메일, 소속, 국가
- 발급일 / 만료일
- 보호 방법 (생체인증 / 비밀번호)
- 일련번호 (hex)
- SHA-256 지문

### 인증서 내보내기

| 버튼 | 출력 형식 | 용도 |
|------|-----------|------|
| `.pkis-cert 저장` | DER 바이너리 | 다른 사람에게 공개키 전달 |
| `PEM 저장` | Base64 텍스트 | OpenSSL 등 호환 도구 사용 |
| `QR 공유` | QR 이미지 | 폰으로 스캔하여 연락처 추가 |
| `카드 공유` | PNG 이미지 | SNS/메신저 공유용 카드 이미지 |

### 신원 삭제

**위험 구역** 섹션의 [신원 삭제] 버튼.
삭제 시 개인 키와 인증서가 **영구 삭제**됩니다. 되돌릴 수 없습니다.

## 6. 파일 서명

사이드바 → **서명** (또는 `/file/sign`)

### 서명 절차

1. **파일 선택**: 서명할 파일을 드래그&드롭하거나 클릭하여 선택
2. **키 잠금 해제**: 생체인증 또는 비밀번호 입력
   - 생체인증+비밀번호 백업이 모두 있는 경우: 상단 토글로 방법 선택
3. **[서명하기]** 클릭
4. `.pkis-sig` 파일 자동 다운로드 + 파일 목록에 저장

### 출력 파일

- 형식: CMS SignedData (DER 인코딩)
- 서명 방식: **Detached signature** — 원본 파일 내용은 포함되지 않음
- 파일명: `원본파일명.pkis-sig`

> 서명 검증 시 **원본 파일과 `.pkis-sig` 파일 모두 필요합니다.**

### 서명에 포함되는 정보

- 서명자 인증서 전체
- 서명 시각 (UTC)
- 원본 파일 SHA-256 다이제스트
- ECDSA P-256 / SHA-256 서명값

---

## 7. 서명 검증

사이드바 → **파일** → 파일 항목의 [검증] 버튼, 또는 `/file/verify`

### 검증 절차

1. **서명 파일** (`.pkis-sig`) 드래그&드롭 또는 클릭 선택
2. **원본 파일** 드래그&드롭 또는 클릭 선택
3. **[서명 검증]** 클릭

### 결과 표시

| 결과 | 의미 |
|------|------|
| ✅ 서명 유효 | 서명이 올바르고 파일이 변조되지 않음 |
| ❌ 서명 무효 | 파일이 변조되었거나 잘못된 원본 파일 |
| ⚠️ 만료/폐지된 신원 | 서명 자체는 유효하나, 서명자의 신원이 갱신되었음 (구버전 인증서로 서명된 파일) |
| 🔴 검증 오류 | 파일 파싱 오류 등 기술적 문제 |

검증 성공 시 표시 정보:
- 서명자 이름
- 서명 시각
- 원본 파일 SHA-256

## 8. 파일 암호화 / 복호화

사이드바 → **암호화** (또는 `/file/encrypt`)

### 탭 구성

화면 상단 탭으로 **암호화** / **복호화** 전환

---

### 8-1. 암호화

1. **암호화할 파일** 선택 (드래그&드롭 또는 클릭)
2. **수신자 선택**
   - 기본: 나(자기 자신) 포함
   - 연락처 목록에서 추가 선택 가능
3. **[암호화하기]** 클릭
4. `.pkis` 파일 자동 다운로드 + 파일 목록에 저장

> 선택된 수신자 모두 각자의 개인 키로 복호화 가능합니다.

#### 암호화 방식

- 알고리즘: CMS EnvelopedData + ECDH P-256 키 합의 + AES-256-GCM
- 각 수신자별로 별도의 키 캡슐화(Key Agreement) 수행
- 파일명: `원본파일명.pkis`

---

### 8-2. 복호화

1. `.pkis` 암호화 파일 선택 (드래그&드롭 또는 클릭)
2. **키 잠금 해제**: 생체인증 또는 비밀번호 입력
3. **[복호화하기]** 클릭
4. 원본 파일 자동 다운로드

#### 주의사항

- 자신이 수신자 목록에 포함된 파일만 복호화 가능
- ⚠️ 경고 표시: 현재 신원이 갱신된 경우, 갱신 전 인증서로 암호화된 파일임을 알림
  - 키 유지 갱신 시: 같은 개인 키를 사용하므로 계속 복호화 가능
  - 새 키 발급 시: 이전 키로 암호화된 파일은 복호화 불가

## 9. 연락처 관리

사이드바 → **연락처** (또는 `/contacts`)

### 연락처 추가 방법

| 방법 | 설명 |
|------|------|
| 파일 가져오기 | `.pkis-cert` 파일 선택 또는 드래그&드롭 |
| QR 스캔 | 상대방의 신원 QR 코드 스캔 (카메라 권한 필요) |
| 텍스트 붙여넣기 | PEM 형식 인증서 직접 입력 |

### 표시 정보

- 이름, 이메일, 소속
- 인증서 만료일
- 신뢰 수준 (자기 자신 / 알려진 / 검증됨)
- SHA-256 지문 (앞 4바이트)

### 연락처 활용

- 파일 암호화 시 수신자로 선택 가능
- 서명 검증 시 서명자 자동 매칭 (지문 비교)

## 10. 파일 목록

### 열 구성

| 열 | 설명 |
|----|------|
| 이름 | 출력 파일명 (원본 파일명 표시) |

| 삭제 | 기록 삭제 (원본 파일에 영향 없음) | 기록 삭제 |

## 11. 신원 갱신 및 만료 신원

신원 페이지 → **[신원 갱신]** 버튼

### 갱신 방식 선택

#### 키 유지 갱신 (권장)
- 기존 개인 키를 그대로 사용
- 인증서 유효기간만 3년 연장
- 기존에 서명한 파일의 검증 가능성 유지
- 기존에 암호화된 파일 계속 복호화 가능

#### 새 키 발급
- 완전히 새로운 키 쌍 + 인증서 생성
- 보안 사고 발생 시 또는 주기적 갱신 시 사용
- **이전 키로 암호화된 파일은 복호화 불가**

### 갱신 절차

1. 갱신 방식 선택
2. 현재 키 인증 (생체인증 또는 비밀번호)
3. 새 키 재봉인 (자동 수행)

### 만료 신원 관리

갱신 시 기존 인증서는 **만료 신원**으로 자동 보관됩니다.

- 신원 페이지 하단 "만료된 신원" 섹션에서 확인
- 만료 신원 정보: 이름, 지문, 만료일, 폐지일

#### 만료 신원 감지

파일 서명 검증 또는 파일 복호화 시:
- 서명에 사용된 인증서 지문이 만료 신원과 일치하면 **⚠️ 경고** 표시
- 서명 자체는 유효하나 서명자의 신원이 이미 갱신되었음을 알림

## 12. PWA 설치

설치하면 주소창 없이 네이티브 앱처럼 실행됩니다.

### iPhone / iPad (Safari)

1. Safari에서 앱 주소 접속
2. 하단 **공유 버튼** (□↑) 탭
3. **"홈 화면에 추가"** 탭
4. 이름 확인 후 **[추가]** 탭
5. 홈 화면에 **KeyID** 아이콘 생성

### Android (Chrome)

1. Chrome에서 앱 주소 접속
2. 앱 내 하단 **"앱으로 설치"** 배너의 [설치하기] 탭
3. 또는 Chrome 메뉴(⋮) → **"홈 화면에 추가"**

### 데스크탑 (Chrome / Edge)

1. 앱 주소 접속
2. 주소창 오른쪽 **설치 아이콘** (⊕) 클릭
3. 또는 브라우저 메뉴 → **"KeyID 설치"**

#### 데스크탑 PWA 특징

- 독립 창으로 실행 (주소창, 탭 없음)
- **Window Controls Overlay**: 타이틀바 영역에 앱 로고 표시, 드래그로 창 이동
- 왼쪽 아이콘 레일 사이드바로 빠른 네비게이션
- 아이콘 위에 마우스를 올리면 메뉴 이름 툴팁 표시

## 13. 기술 참고사항

### 암호화 알고리즘

| 용도 | 알고리즘 |
|------|---------|
| 키 쌍 생성 | ECDSA P-256 |
| 파일 서명 | ECDSA P-256 / SHA-256 |
| 키 합의 (암호화) | ECDH P-256 |
| 콘텐츠 암호화 | AES-256-GCM |
| 키 봉인 (비밀번호) | PBKDF2 (SHA-256, 100,000회) → AES-256-GCM |
| 키 봉인 (생체인증) | WebAuthn PRF → AES-256-GCM |
| 인증서 지문 | SHA-256 |

### 데이터 저장 위치

모든 데이터는 브라우저의 **IndexedDB** (`keyid-v1`)에 저장됩니다.

| 스토어 | 내용 |
|--------|------|
| `identity` | 자신의 신원 (봉인된 개인 키 + 인증서) |
| `contacts` | 연락처 (공개 인증서) |
| `files` | 서명/암호화된 파일 기록 + 데이터 |
| `expired_identities` | 갱신 전 만료 신원 기록 |

### 브라우저 호환성

| 기능 | Chrome | Firefox | Safari | Edge |
|------|--------|---------|--------|------|
| 기본 기능 | ✅ | ✅ | ✅ | ✅ |
| WebAuthn PRF | ✅ 116+ | ✅ 119+ | ✅ 16+ | ✅ 116+ |
| PWA 설치 | ✅ | ❌ | ✅ (홈화면) | ✅ |
| Window Controls Overlay | ✅ | ❌ | ❌ | ✅ |

## 14. 파일 포맷 설명

### `.pkis-cert` — 인증서

- 형식: DER 인코딩 X.509 v3 인증서
- 공개키 + 신원 정보 포함
- 상대방에게 공유하는 파일 (개인 키 포함 안 됨)

### `.pkis-sig` — 서명 파일

- 형식: CMS ContentInfo → SignedData (DER)
- Detached 서명 (원본 파일 미포함)
- 서명자 인증서 + 서명 시각 + 파일 다이제스트 포함
- 검증 시 원본 파일과 함께 제공 필요

### `.pkis` — 암호화 파일

- 형식: CMS ContentInfo → EnvelopedData (DER)
- 원본 파일 내용이 AES-256-GCM으로 암호화됨
- 각 수신자별 키 캡슐화(KeyAgreeRecipientInfo) 포함

### `.pkis-req` — 서명 요청 (향후 기능)

- 서명 요청서 파일

### `.pkis-key` — 키 파일 (향후 기능)

- 봉인된 개인 키 내보내기/가져오기

## 15. 문제 해결

### 페이지가 스피너만 표시되고 멈춤

**원인**: IndexedDB 스키마 업그레이드 충돌

**해결**:
1. 브라우저 개발자 도구 → Application → IndexedDB → `keyid-v1` 삭제
2. 페이지 새로 고침
3. 신원 재생성 필요

> ⚠️ 개인 키가 삭제됩니다. 인증서를 먼저 내보내 두세요.

---

### "Failed to execute 'transaction' on 'IDBDatabase'"

**원인**: DB 버전이 3이지만 `expired_identities` 스토어 미생성

**해결**:
- 앱을 최신 버전(v4 이상)으로 업데이트하면 자동 해결
- 또는 위 IndexedDB 삭제 후 재생성

---

### 생체 인증이 작동하지 않음

**원인/확인사항**:
- HTTPS 또는 localhost에서만 WebAuthn 동작
- 기기/브라우저가 WebAuthn PRF 확장을 지원해야 함
- 지원 여부 확인: Chrome 116+, Safari 16+, Firefox 119+

**해결**: 백업 비밀번호 사용 (신원 생성 시 설정한 경우)

---

### 복호화 실패 — "key.algorithm does not match"

**원인**: 내부 키 임포트 방식 오류 (이전 버전 버그)

**해결**: 앱 최신 버전에서 자동 해결됨 (ECDH로 재임포트)

---

### 폰에서 주소창이 사라지지 않음

**원인**: PWA로 설치되지 않고 브라우저에서 직접 접속 중

**해결**: [12. PWA 설치](#12-pwa-설치) 절차에 따라 홈 화면에 추가

---

### QR 코드 스캔이 안 됨

**원인**: 카메라 접근 권한 미허용 또는 HTTPS 미사용

**해결**:
- 브라우저 설정에서 카메라 권한 허용
- HTTPS 주소로 접속 (cloudflared 터널 사용)

---
